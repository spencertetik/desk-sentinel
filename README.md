# Desk Sentinel

A fully-local desk-health tracker that watches your posture through an IP camera
you already own, gently coaches you to sit up and take breaks, and keeps
long-term analytics on your work habits — **all on your own machine**. No cloud,
no accounts, no video ever leaves your computer.

Point it at any RTSP camera aimed at your desk. It uses
[MediaPipe Pose](https://developers.google.com/mediapipe) to read your posture,
shows a live mirror with a skeleton overlay in your browser, and (optionally)
speaks short reminders out loud when you slouch or have been sitting too long.

> Built for a UniFi Protect camera, but works with any camera that exposes an
> RTSP stream (Reolink, Wyze with the RTSP firmware, Amcrest, a phone running an
> RTSP app, etc.).

---

## What it does

- **Live posture mirror** — your camera feed in the browser with a pose skeleton
  drawn on top, labeled good / forward-head / slouching in real time.
- **Coach** — debounced, spoken nudges via the macOS `say` voice: "you've been
  sitting an hour, take a break", "posture's slipping", plus a morning brief and
  an end-of-day wind-down.
- **Presence insights** — a dashboard of when you were at your desk, session
  lengths, break rhythm, and trends over days/weeks.
- **Active-work detection** — tells "actively working" apart from "sitting idle"
  by watching motion in the keyboard region of the frame.
- **Voice agent (optional)** — push-to-talk or a dashboard mic button; ask it
  questions about your day and it answers out loud. Runs on local
  [faster-whisper](https://github.com/SYSTRAN/faster-whisper) for speech-to-text
  and a local [Ollama](https://ollama.com) model for the answer. No API keys.
- **Long-term memory** — everything is logged to a local SQLite database with
  daily rollups and rotating backups, so the analytics survive restarts and grow
  over time.
- **Always-on (optional)** — a macOS launchd setup keeps it running and
  self-healing across crashes and reboots (see [`deploy/`](deploy/)).

Everything runs locally. The only outbound network calls are optional: the
morning brief can fetch weather and AI-news headlines, and that's it.

## Requirements

- **macOS** (the spoken coach uses the built-in `say`; everything else is
  cross-platform, but the voice nudges are Mac-specific).
- **Python 3.11**
- **An RTSP camera** pointed at your desk (you'll need its RTSP URL).
- *(Optional)* [Ollama](https://ollama.com) installed and a model pulled
  (default `llama3.1:8b`) if you want the conversational voice agent.

## Quick start

```bash
# 1. clone + enter
git clone https://github.com/<your-username>/desk-sentinel.git
cd desk-sentinel

# 2. create a virtual environment + install
python3.11 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt

# 3. tell it about your camera
#    open config.yaml and set rtsp_url to your camera's RTSP URL
#    (look in your camera/NVR app — UniFi Protect, Reolink, etc.)

# 4. calibrate your "neutral" sitting posture (sit up straight, run this once)
python -m sentinel.app --calibrate

# 5. run it
python -m sentinel.app
```

Then open the dashboard at **http://localhost:8088**.

### Configuring your camera

Open [`config.yaml`](config.yaml) and set `rtsp_url`. Examples:

```yaml
stream:
  rtsp_url: rtsp://USERNAME:PASSWORD@192.168.1.50:554/stream1   # most cameras
  # rtsp_url: rtsp://192.168.1.1:7447/<alias>                   # UniFi Protect
```

If you're not sure of your camera's RTSP URL, check its app/NVR settings — most
expose it under "Advanced", "RTSP", or "Stream". UniFi Protect users can enable
RTSP per-camera in the device settings.

### Calibration

Posture is measured relative to *your* neutral seated position (the camera can't
see your hips behind the desk, so it tracks how far your head drops from
upright). Sit up the way you'd like to sit, run `python -m sentinel.app
--calibrate` once, and it saves a `baseline.json`. If posture detection feels too
strict or too loose, nudge `thresholds.head_drop_margin` in `config.yaml` (bigger
= more forgiving) and re-run.

## The voice agent (optional)

The dashboard mic button and push-to-talk hotkey let you *talk* to Desk Sentinel.
It's entirely local:

1. Install [Ollama](https://ollama.com) and pull the model:
   ```bash
   ollama pull llama3.1:8b
   ```
2. Make sure Ollama is running (`ollama serve`, or just launch the app).
3. Use the mic button on the dashboard, or the hotkey (default `⌘+⌥+D`).

Speech-to-text runs on faster-whisper locally; the answer is generated by your
local Ollama model and spoken back through the Mac voice. Configure models and
the hotkey under `voice:` in `config.yaml`.

## Always-on supervision

To keep Desk Sentinel running across crashes and reboots on macOS, see
[`deploy/README.md`](deploy/README.md) — it sets up a launchd LaunchAgent with a
self-healing capture watchdog so a frozen feed automatically restarts a clean
instance.

## Configuration

Everything is in [`config.yaml`](config.yaml), grouped by area: `stream`
(camera), `thresholds` (posture sensitivity), `nudges` (coach timing + work
hours + morning brief / wind-down), `presence`, `activity` (active-work
detection), `voice`, and `backup`. Each block is commented. Sensible defaults are
provided for everything except the camera URL.

Your data lives at `~/.desk-sentinel/desk_sentinel.db` and backups (if enabled)
go to `~/desk-sentinel-backups` — point `backup.dest_dir` at an external drive or
NAS for off-machine durability.

## How it fits together

```
RTSP camera ──▶ capture (reconnecting, drains at native rate)
                   │
                   ▼
            MediaPipe Pose ──▶ posture metrics ──▶ classify (good/forward/slouch)
                   │                                     │
                   ▼                                     ▼
            skeleton overlay                       event engine ──▶ spoken coach
                   │                                     │
                   └──────────▶ FastAPI server ◀─────────┘
                                     │
                            web dashboard (:8088)
                                     │
                                 SQLite store ──▶ daily rollups + backups
```

- `sentinel/capture.py` — self-healing RTSP reader (socket timeout, reconnect,
  freshness-based health).
- `sentinel/pose.py`, `metrics.py`, `classify.py` — pose → posture label.
- `sentinel/events.py`, `nudges.py` — debounced coach + spoken reminders.
- `sentinel/presence.py`, `activity.py` — presence + active-work analytics.
- `sentinel/voice/` — local STT + Ollama voice agent.
- `sentinel/store.py`, `backup.py` — SQLite persistence, rollups, rotating
  backups.
- `sentinel/server.py`, `web/` — FastAPI API + browser dashboard.

---

## Privacy

Desk Sentinel is local-first by design. Video frames are processed in memory and
never written to disk or sent anywhere. The only data persisted is numeric
posture/presence metrics in a local SQLite file. The morning brief's optional
weather/news lookups are the sole outbound calls, and you can disable them in
`config.yaml`.
